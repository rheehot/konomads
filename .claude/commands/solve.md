# GitHub Issue Solver Custom Command

## Name
`solve` - GitHub 이슈 분석 및 해결 계획 수립

## Description
GitHub 이슈 번호를 입력하면 이슈를 가져와서 해결하기 위한 상세한 계획을 세우고 검토받습니다.

## Usage
```
/solve $ARGUMENT
```

## Arguments
- `$ARGUMENT`: GitHub 이슈 번호 (예: 42)

## System Prompt

당신은 GitHub 이슈 해결 전문가입니다. 입력받은 이슈 번호로 이슈를 조회하고, 체계적으로 해결 계획을 수립한 후 사용자에게 검토를 받으세요.

---

## 작업 프로세스

### 1. 이슈 조회 (먼저 실행)
먼저 다음 bash 명령을 실행하여 이슈 정보를 가져오세요:
```bash
gh issue view $ARGUMENT --json title,body,labels,state,number,author,url,comments
```

### 2. 이슈 분석
가져온 이슈 정보를 바탕으로 다음을 분석하세요:
- 이슈의 핵심 내용
- 현재 상태 (open/closed)
- 관련 라벨
- 코멘트 내용 (있는 경우)

### 3. 해결 계획 수립
다음 형식으로 해결 계획을 작성하세요.

## 해결 계획

### 🎯 목표
생활비 섹션의 숫자가 모든 화면 크기에서 정상적으로 표시되도록 수정

### 🔍 분석
- 현재 grid-cols-3으로 고정되어 있어 모바일에서 공간 부족
- 숫자 포맷팅(만 단위)이 긴 경우 텍스트가 넘침

### ✅ 작업 계획

#### Phase 1: 문제 확인 (선행: 없음)
- [ ] **작업 1.1**: 현재 레이아웃 확인
  - 📥 입력: app/cities/[slug]/page.tsx
  - 📤 출력: 문제 지점 파악
  - ✅ 완료 기준: CSS 클래스와 레이아웃 구조 파악
  - ⏱️ 예상 시간: 5분

#### Phase 2: 수정 (선행: Phase 1)
- [ ] **작업 2.1**: 반응형 grid 적용
  - 📥 입력: 현재 코드
  - 📤 출력: grid-cols-1 md:grid-cols-3 적용
  - ✅ 완료 기준: 모바일에서 1열로 표시
  - ⏱️ 예상 시간: 10분

- [ ] **작업 2.2**: 숫자 표시 영역 수정
  - 📥 입력: 생활비 카드 코드
  - 📤 출력: text-overflow 또는 truncate 적용
  - ✅ 완료 기준: 긴 숫자도 잘리지 않음
  - ⏱️ 예상 시간: 10분

#### Phase 3: 검증 (선행: Phase 2)
- [ ] **작업 3.1**: 다양한 화면 크기 테스트
  - 📥 입력: 수정된 페이지
  - 📤 출력: 테스트 결과
  - ✅ 완료 기준: 375px, 768px, 1280px에서 정상 표시
  - ⏱️ 예상 시간: 10분

- 금액 단위(만) 표시 로직은 변경하지 않음
